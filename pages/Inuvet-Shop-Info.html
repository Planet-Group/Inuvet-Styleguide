inuvet Shop — Dokumentation

Dieses Dokument beschreibt das Mockup der regulären inuvet-Shop-Startseite und insbesondere das Praxis-Opt-in-Pop-up. Zielgruppe sind IT und Marketing. Das Mockup ist unter Inuvet-Shop.html einsehbar.

Überblick — Startseite

Die Startseite zeigt den regulären inuvet-Shop ohne Tierarzt-Empfehlungs-Branding. Aufgebaut ist sie aus folgenden Sektionen:

  1. Hauptnavigation — Announcement Bar, Mockup/Dokumentation-Links, Logo, Suche und Warenkorb-Icon (ohne Drawer)
  2. Hero-Section — Variante 1 mit Lottie-Animation (Hero_Animation_01.json); Höhe ergibt sich aus dem nativen Seitenverhältnis der JSON
  3. USP-Kacheln — vierspaltiges Raster mit Lottie-Icons (Beste Expertise, Langzeitgabe, Unterstützung, Lieferzeit)
  4. Produkt-Collection — vierspaltiges Produktraster (.tile-grid --cols-4)
  5. Footer — Newsletter, Shop-Links, Rechtliches (Layout wie Tierarzt-Empfehlung)

Zusätzlich erscheint beim Seitenaufruf das Praxis-Opt-in-Pop-up (siehe unten).

Praxis-Opt-in-Pop-up

Das Pop-up informiert Tierarztpraxen über die neue Funktion »Tierarzt-Empfehlung« und fragt, ob Inuvet die Praxis kontaktieren darf, wenn ein Tierbesitzer eine Empfehlung wünscht.

Zielgruppe

Die Ansprache richtet sich an Tierarztpraxen (Plural „ihr“, „euch“) — nicht an Endkunden/Tierbesitzer.

Wann erscheint das Pop-up?

  • Bei jedem Aufruf der Startseite (Inuvet-Shop.html)
  • Automatisch ca. 400 ms nach dem Laden der Seite (kurze Verzögerung, damit Hero und Seite zuerst sichtbar sind)
  • Es gibt keine Speicherung der Entscheidung — beim nächsten Seitenaufruf erscheint das Pop-up erneut

Verhalten

  • Das Pop-up blockiert die Seite (Hintergrund abgedunkelt, Scrollen der Seite gesperrt)
  • Es kann nicht per X, Klick auf den Hintergrund oder Escape geschlossen werden
  • Der Nutzer muss eine der beiden Optionen wählen

Buttons

ButtonStyleReaktion
Ja, gerne! Primary (.btn --primary --full) Pop-up schließt sich. Grüne Toast-Meldung unten rechts: „Danke — wir melden uns bei euch!"
Nein Ghost (.btn --ghost --full) Pop-up schließt sich. Keine weitere Meldung.

Design-Pattern

Das Pop-up nutzt die globale Komponente .shop-modal aus dem Styleguide (C.10 — Bestätigungs-Dialog). Gleiches Pattern wie das Freigabe-Willkommen-Pop-up auf der Tierarzt-Empfehlung-Startseite.

Technische Umsetzung (Mockup)

Aktuell ist das Pop-up ein reines Frontend-Mockup — keine Daten werden gespeichert oder an ein Backend übermittelt. Die Auswahl (Ja/Nein) hat keine Auswirkung auf CRM, E-Mail oder Praxis-Stammdaten.

// Beim Laden: Pop-up nach 400 ms öffnen
document.addEventListener('DOMContentLoaded', () => {
  setTimeout(openPracticeOptIn, 400);
});

// Schließen nur über Button-Klick
function closePracticeOptIn(accepted) {
  overlay.classList.remove('--open');
  document.body.style.overflow = '';
  if (accepted) showToast('Danke — wir melden uns bei euch!');
}

Offene Fragen für Produktion

Für die Live-Umsetzung in Shopify sollten IT und Marketing gemeinsam klären:

  • Wann anzeigen? Nur für erkannte Praxen? Nur einmal pro Praxis? Nur wenn noch kein Opt-in vorliegt?
  • Speicherung — Wo wird die Antwort (Ja/Nein) persistiert? Praxis-Stammdaten, CRM, Shopify Metafields?
  • Backend-Anbindung — Was passiert bei „Ja"? Automatische E-Mail an Innendienst? CRM-Task? Update im Praxis-Portal?
  • Opt-out — Kann eine Praxis die Entscheigung später widerrufen? Wo im Portal?
  • Seiten-Scope — Nur Startseite oder auch andere Shop-Seiten für eingeloggte Praxis-Accounts?

Mockup-Hinweis: Im Styleguide-Mockup erscheint das Pop-up bei jedem Reload absichtlich immer wieder — damit IT und Marketing den Flow jederzeit testen können. In Produktion wird typischerweise nach der ersten Entscheidung nicht erneut gefragt.
